# 金星の見え方シミュレーター

太陽の周りを金星が公転するようすと、地球から見た金星の「満ち欠け」「見える方角・高さ」を
同時に確認できる、授業用のシンプルなシミュレーターです。
HTML1ファイルのみで動作し、iPad・PC・スマートフォンのブラウザでそのまま開けます。

## 使ってみる

以下のURLをブラウザで開くだけで使えます。

```
https://nk-1994.github.io/venus-simulator/
```

インターネット環境がなくても、`index.html` をダウンロードして
ブラウザで直接開けば動作します。

## できること

### 1. 上から見た図（太陽系を俯瞰する視点）
- 太陽を中心に、地球は下側に固定、金星だけが公転します。
- 金星・地球とも「太陽の光が当たる側だけ明るい」半月状に描かれるため、
  満ち欠けの原理（光る側は常に太陽の方向）がそのまま見た目に表れます。
- 金星は指やマウスでドラッグして自由に動かせます。

### 2. 地球からの視点（観測者モード）
- 「日没30分後の西の空」「日の出30分前の東の空」を再現します。
- 金星の位置に応じて、宵の明星／明けの明星が自動で切り替わります。
- 太陽に近すぎて見えない期間（内合・外合付近）は、その旨が画面に表示されます。
- 再生すると1日ごとに観測記録の点が蓄積され、
  金星が太陽から離れて登り、最大離角で折り返して戻っていく
  **曲線的な軌跡**が画面に残っていきます（15日ごとに、その日の満ち欠けの形も表示）。

### 3. 左上ののぞき窓
- 地球から見た金星の満ち欠けの形を、常時表示します。
- 地球ー金星間の距離に応じて見かけの大きさも変化します
  （内合付近は大きな三日月、外合付近は小さな丸）。

### 操作パネル
| 操作 | 内容 |
|---|---|
| ▶ / ⏸ | 公転アニメーションの再生・一時停止 |
| 速度スライダー | 公転の速さを調整 |
| 🔭 視点切り替え | 「上から見た図」⇔「地球からの視点」を切り替え |
| 🧹 記録を消す | 地球視点で蓄積された観測記録をリセット |
| ドラッグ | 金星を直接動かして好きな配置を作れる（観測記録は自動リセット） |

## 授業での使い方の例

1. **満ち欠けの原理**：上から見た図で金星をドラッグしながら、左上ののぞき窓の形の変化を確認する。
2. **見える時間帯・方角**：地球視点に切り替え、宵の明星／明けの明星の切り替わりと
   太陽との位置関係を確認する。
3. **継続観察との接続**：「日没から30分後に観察しよう」という実際の観察指示と
   画面の基準時刻を揃えているため、生徒が家庭で記録した内容とシミュレーターの
   軌跡を突き合わせることができる。
4. **ガリレオの観測**：満ち欠け（形の変化）と見かけの大きさの変化を同時に示すことで、
   天動説では説明できない配置（金星が満月状になる場面）があることに気づかせる。

## モデルの簡略化について（授業で触れる場合の注意）

- 金星・地球ともに**円軌道**として計算しています（実際はわずかに楕円）。
- 地球視点の曲線は、黄道傾斜角23.4°・観測地の緯度（所沢:35.8°N）を用いた
  簡易的な天文計算により求めていますが、**日付は実際の暦とは連動していません**
  （春分からの経過日数を内部的に進めているだけの、定性的な再現です）。
- 恒星を背景にした「順行・逆行」は表現していません。これは地球自身の公転を
  考慮しないと原理的に表現できないため、今回は「太陽からの離れ具合（離角）」と
  「満ち欠け」の対応を主眼にしたモデルにしています。

## 公開設定（GitHub Pages）

このリポジトリは GitHub Pages で公開済みです。設定は以下の通りです。

- **Settings → Pages → Build and deployment → Source**：`Deploy from a branch`
- **Branch**：`main` / `/ (root)`
- 公開ページのエントリーファイルは `index.html`（GitHub Pagesが既定で探すファイル名）

### 更新時の反映について

`index.html` を更新してコミットすると、自動的に再デプロイされます。
反映まで数分かかることがあるので、変更が見えないときはブラウザの再読み込み
（キャッシュが強い場合はシークレットウィンドウでの確認）を試してください。

### デプロイが失敗する場合

**Settings → Actions**（または画面下部のコミットステータス）に赤い✕が出た場合は、
まず **Actions タブ** を開き、失敗した実行のログを確認してください。

- `Error: Deployment failed, try again later.` というメッセージが出ている場合、
  多くはコード側の問題ではなく **GitHub Pages側の一時的な不調** です。
  該当の実行を開き「Re-run failed jobs」で再実行すると、それだけで解決することが
  よくあります。
- 何度か再実行しても直らない場合は、Settings → Pages で Source を一度 `None` に
  戻して保存し、再度 `Deploy from a branch` / `main` / `root` に設定し直すと
  改善することがあります。
- リポジトリ内に古いファイル（例：以前使っていた `venus-simulator.html`）が
  残っていないか、ファイル一覧で確認してください。
